# Phase 4b Fixes Summary — 2026-06-24

## Applied from Sonnet review

**Fix 1 — Outer loop guards changed from `<= 0` to `< MIN_BLOCK_MINUTES`**
- File: `server/src/lib/scheduler.ts`, lines 104 and 140
- Was: `if (remainingMinutes <= 0) break`
- Now: `if (remainingMinutes < MIN_BLOCK_MINUTES) break`
- Reason: when `remainingMinutes` drops below 30 (e.g., after placing a partial block in a narrow gap), the old guards allowed the outer loops to spin through all remaining days/gaps — each inner `while` would hit the `blockMin < 30` guard and immediately break, wasting O(D×G) iterations without placing any blocks. The new guard exits early and correctly.

## Applied from Opus review

**Fix 2 — `while` condition and dead inner guard removed**
- File: `server/src/lib/scheduler.ts`, line 143
- Was: `while (remainingMinutes > 0)` + `if (blockMin < MIN_BLOCK_MINUTES) break` (line 148)
- Now: `while (remainingMinutes >= MIN_BLOCK_MINUTES)` (line 148 removed)
- Reason: after Fix 1, lines 140 and 145 guarantee `remainingMinutes >= 30` and `availableMin >= 30` before reaching `Math.min(...)`. Therefore `blockMin >= 30` always, making the old line 148 guard structurally unreachable dead code. Changing the `while` condition to `>= MIN_BLOCK_MINUTES` makes the intent self-documenting and naturally eliminates the dead branch.
