# Phase 1a — Opus Code Review
**Date:** 2026-06-23  
**Phase:** 1 — Foundation  
**Sub-phase:** 1a — Monorepo & Tooling

## Findings

### Finding 1 — server/src/index.ts:15 (HIGH) — CONFIRMED — FIXED
**Issue:** `app.listen()` had no `.on('error', ...)` handler. An EADDRINUSE (or EACCES) event on the underlying net.Server would be an unhandled EventEmitter error, crashing the process with no useful diagnostic message.  
**Fix applied:** Captured the return value of `app.listen()` into `server` and attached `server.on('error', ...)` with specific EADDRINUSE messaging and `process.exit(1)`.

### Finding 2 — server/src/index.ts:6 (MEDIUM) — CONFIRMED — FIXED
**Issue:** `process.env.PORT ?? 3000` produces `string | number`. TypeScript won't flag `PORT === 3000` (always false when env is set) or numeric arithmetic producing `NaN`/concatenated strings.  
**Fix applied:** Changed to `Number(process.env.PORT) || 3000` — always `number`.

### Finding 3 — server/package.json (MEDIUM) — CONFIRMED — FIXED
**Issue:** `jest`, `ts-jest`, and `@types/jest` absent from devDependencies. CLAUDE.md lists `npx jest` as a first-class command and the quality gate requires Jest before every commit. Test files would fail to compile from Phase 2a onward.  
**Fix applied:** Added `jest ^29.7.0`, `ts-jest ^29.2.2`, `@types/jest ^29.5.12` to devDependencies; added `"test": "jest"` script; created `jest.config.ts` with `ts-jest` preset.

### Finding 4 — app/src/types/declarations.d.ts (LOW) — PLAUSIBLE — DEFERRED
**Issue:** `declare module '*.module.css'` is overly broad — it makes TypeScript accept CSS module imports in any file, including non-web React Native components where Metro would fail at bundle time. The existing template's `.web.tsx` and `global.css` usages are legitimate, but the ambient declaration could silently mask future incorrect imports.  
**Assessment:** Deferred. The declarations were added to fix Expo 56 template TypeScript errors; scope is appropriate for the current template files. Will revisit if non-web CSS module imports appear in later phases.

### Finding 5 — root package.json (LOW) — CONFIRMED — DEFERRED
**Issue:** `lint:all` uses `&&` — server lint never runs if app lint fails, masking server errors in CI.  
**Assessment:** Deferred. Short-circuit behavior is intentional for developer flow ("fix one thing at a time"). Will add `npm-run-all` or parallel execution if this becomes a CI pain point.

### Finding 6 — server/src/index.ts (LOW) — CONFIRMED — DEFERRED
**Issue:** No `SIGTERM`/`SIGINT` handler. Will cause issues once Prisma is added (Phase 1c) — abrupt kill during a write could leave SQLite WAL inconsistent.  
**Assessment:** Deferred to Phase 1c when Prisma is introduced — graceful shutdown is most meaningful once there's a DB connection to close.

## Verdict: 3 fixes applied, 3 deferred
