# Phase 1b — Fixes Summary (2026-06-23)

## Sonnet fixes applied

1. **auth.ts — crypto.timingSafeEqual**: Replaced plain `!==` token comparison with `Buffer`-based `timingSafeEqual` to prevent timing side-channel attacks.

2. **auth.ts — Startup validation**: Added `if (!process.env.API_BEARER_TOKEN) throw` so the server fails fast on missing config rather than returning 500 on every request.

3. **timezone.ts — IANA validation**: Added `isValidIANATimezone` guard using `Intl.DateTimeFormat` try/catch. Invalid timezone strings fall back to UTC silently instead of propagating to downstream date math where they'd throw RangeError.

## Opus fixes applied

4. **Critical import-ordering fix**: Removed the module-level throw and `TOKEN_BUF` constant from `auth.ts`. TypeScript imports compile to `require()` calls that execute before any code in the importing module, so the original module-level code in `auth.ts` ran before `dotenv.config()` in `index.ts` — the token was always undefined at module load. Fix: `auth.ts` reads the token per-request; startup validation moved to `index.ts` after `dotenv.config()`.

## Not applied (noted)

- Length oracle in `timingSafeEqual` (low severity for single-user local app)
- Deprecated IANA timezone aliases (engine-dependent, acceptable)
- `statusCode` bounds check (no current exploitable path, would be over-engineering per CLAUDE.md)
