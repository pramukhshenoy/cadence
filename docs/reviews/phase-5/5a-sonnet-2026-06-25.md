# Phase 5a Sonnet Review — 2026-06-25

## Findings (ranked most severe first)

1. **HIGH** `app/src/hooks/use-sleep-sync.ts:79` — `useSleepSummary` uses `getQueryData` (snapshot) not `useQuery` (subscription); callers never re-render when sleep sync completes. Dashboard mounts before sync finishes; sleep card never appears.

2. **HIGH** `app/src/hooks/use-sleep-sync.ts:75` — Threshold fields in dependency array re-run the full Health Connect chain (getSdkStatus → init → permission → readRecords) on every settings change. Should split data-fetch effect from quality-derivation.

3. **MEDIUM** `server/src/routes/settings.ts:68` — No cross-field validation that `sleepThresholdHours < goodThresholdHours`. Inverted values collapse the FAIR quality tier: any sleep ≥ sleepThresholdHours (8) is GOOD, no FAIR band possible.

4. **MEDIUM** `app/src/hooks/use-sleep-sync.ts:68` — If `readLastNightSleep()` throws, `.catch` logs but does NOT call `setQueryData(null)`. Cache stays `undefined` indefinitely; dashboard `=== null` checks never fire.

5. **MEDIUM** `app/src/hooks/use-sleep-sync.ts:37` — No "already fetched today" guard; full Health Connect read fires on every app foreground.

6. **LOW** `app/src/lib/health-connect.ts:34` — `initHealthConnect` catch swallows all errors, returning `false` for both "SDK unavailable" and "transient crash". Transient crash silently disables sleep tracking for the session.

7. **LOW** `app/src/hooks/use-sleep-sync.ts:79` — `useSleepSummary` return type is `SleepSummary | null` but `getQueryData` can also return `undefined`; TypeScript won't catch the missing case.

8. **LOW** `app/src/lib/health-connect.ts:69` — `sleepThresholdHours` validation accepts NaN (`typeof NaN === 'number'` and `NaN <= 0` is `false`). Low production risk (JSON can't encode NaN) but fixable with `Number.isFinite()`.
