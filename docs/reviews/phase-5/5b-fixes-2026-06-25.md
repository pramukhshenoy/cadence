# Phase 5b Fixes — 2026-06-25

## Sonnet findings applied (6 total)

1. **`droppedCount` under-counts partial blocks** (`sleep-reschedule.ts`)
   - `Math.floor(45/60) = 0` when 45 min unscheduled. Fixed to `Math.ceil(remainingMinutes / BLOCK_DURATION_MINUTES)` when `remainingMinutes >= MIN_BLOCK_MINUTES`.

2. **DB not stamped before returning plan** (`routes/sleep.ts`)
   - Morning blocks not marked `rescheduled=true`. Added `prisma.focusBlock.updateMany` call before `computeReschedule` so a client crash leaves a recoverable signal.

3. **`sessionCount: null` → 400** (`routes/sleep.ts`)
   - Inconsistent with `deepSleepHours: null` behavior. Added `&& sessionCount !== null` to the validation guard.

4. **Constants duplicated** (`sleep-reschedule.ts`, `scheduler.ts`)
   - `MIN_BLOCK_MINUTES` and `BLOCK_DURATION_MINUTES` declared in both files. Exported from `scheduler.ts`, imported in `sleep-reschedule.ts`.

5. **`utcToLocalDateStr` / `utcToLocalHour` duplicated** (`routes/sleep.ts`, `scheduler.ts`)
   - Local `utcToLocalParts` helper duplicated logic already in `scheduler.ts`. Exported `utcToLocalDateStr` and new `utcToLocalHour` from `scheduler.ts`; removed local helper from `sleep.ts`.

6. **Cross-field validation missing** (`routes/settings.ts`)
   - `PATCH /api/settings` accepted `sleepThresholdHours >= goodThresholdHours`. Added cross-field check: `sleepThresholdHours < goodThresholdHours` enforced, handling partial updates by reading current DB value.
