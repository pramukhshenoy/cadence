# Phase 5a Fixes Summary ‚Äî 2026-06-25

## Sonnet Findings ‚Üí Applied

1. **HIGH ‚Äî useSleepSummary not reactive**: Changed from `getQueryData` snapshot to `useQuery` with `queryFn: skipToken`. Callers now re-render reactively when data arrives via `setQueryData`.

2. **HIGH ‚Äî HC re-read on settings change**: Split single effect into two:
   - Effect 1: Health Connect read (depends only on `queryClient`)
   - Effect 2: Quality re-derivation on threshold changes (reads cache, no HC call)
   Threshold values accessed via `useRef` in Effect 1 to avoid re-triggering the expensive HC chain.

3. **MEDIUM ‚Äî Cross-field validation missing**: Added `sleepThresholdHours < goodThresholdHours` enforcement to `server/src/routes/settings.ts` PATCH handler. Handles partial updates by fetching current DB value for the missing field, with optimization to skip the DB read when both fields are present.

4. **MEDIUM ‚Äî Error handler leaves cache undefined**: `.catch` in Effect 1 now only logs ‚Äî does NOT write any cache key. This keeps the `undefined` guard clear for retries on next mount.

5. **MEDIUM ‚Äî No "already fetched" guard**: Introduced `RAW_SLEEP_KEY` as the guard sentinel (only written on successful HC read). Effect 1 checks `getQueryData(RAW_SLEEP_KEY) !== undefined` before running.

6. **LOW ‚Äî initHealthConnect swallows errors**: Removed try/catch in `initHealthConnect` ‚Äî errors now propagate to Effect 1's `.catch` for proper session-level error handling.

7. **LOW ‚Äî useSleepSummary missing undefined in return type**: Changed return type to `SleepSummary | null | undefined` to cover the pre-sync state.

8. **LOW ‚Äî Number.isFinite() missing**: Added `Number.isFinite()` check to `sleepThresholdHours` and `goodThresholdHours` validation in the PATCH handler.

## Opus Findings ‚Üí Applied

1. **CRITICAL ‚Äî useSleepSummary queryFn poisons cache**: `queryFn: () => null` was executing on first mount, setting `SLEEP_SUMMARY_KEY = null`, permanently blocking Effect 1's guard. Fixed with `queryFn: skipToken` ‚Äî TanStack Query never executes it, but still subscribes reactively.

2. **CRITICAL ‚Äî Race condition between Effect 1 and Effect 2**: Effect 2 runs synchronously before Effect 1's async HC read completes (finds `RAW_SLEEP_KEY=undefined`, exits). Effect 1 would write `RAW_SLEEP_KEY` later but Effect 2 never re-runs. Fixed by having Effect 1 write `SLEEP_SUMMARY_KEY` directly using threshold values from refs immediately after the HC read, eliminating the dependency on Effect 2 for initial population.

3. **HIGH ‚Äî Error catch blocked retries permanently**: Catch handler was writing `SLEEP_SUMMARY_KEY=null`, which acted as a guard that prevented any future retry. Fixed: catch only logs, writes no cache keys.

4. **HIGH ‚Äî Guard on wrong key**: Was guarding on `SLEEP_SUMMARY_KEY` which is polluted by multiple sources (queryFn, error handler). Changed guard to `RAW_SLEEP_KEY` which is only ever written on a confirmed successful HC read.

5. **MEDIUM ‚Äî rescheduledCount overwritten on threshold change**: Effect 2 now reads `prev?.rescheduledCount ?? 0` from the existing `SLEEP_SUMMARY_KEY` cache entry before writing the new summary, preserving the count set by phases 5b/5c.

6. **LOW ‚Äî Unnecessary alias locals in Effect 2**: Removed redundant `thresholdHours`/`goodHours` alias variables; TypeScript narrows `sleepThresholdHours` and `goodThresholdHours` directly after the undefined guard.

7. **LOW ‚Äî Gratuitous DB read optimization**: PATCH handler only fetches current settings when one of the two threshold fields is missing from the request body (not when both are present).

8. **LOW ‚Äî Object literal type error (TypeScript 6)**: TypeScript 6 is stricter about matching object literals against union types in generic `setQueryData` calls. Fixed by declaring `const summary: SleepSummary = {...}` before passing to `setQueryData` in both Effect 1 and Effect 2.
