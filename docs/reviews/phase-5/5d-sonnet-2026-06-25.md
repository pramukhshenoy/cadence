## Findings applied

1. **settings.tsx:82** — Added validation: if any sleep field is non-empty but NaN, throw before saving. This catches 'abc' typed in a field being silently dropped. The all-empty case (all three cleared) is a no-op with 'Saved ✓', which is acceptable since connection/model settings did save.

2. **settings.tsx:79** — No fix applied. Server-side validation is sufficient for a single-user personal app. Adding client-side cross-field validation would add complexity per CLAUDE.md.

3. **use-sleep-sync.ts:66** — No fix applied. Effect 2 self-corrects within milliseconds. The race window is inherent to the async design.

## Refuted candidates

- **useSleepSync never mounted**: REFUTED — `SleepSyncTrigger` in `_layout.tsx` calls `useSleepSync()` unconditionally.
- **sleepInitialized blocks re-sync after save**: REFUTED — server stores exact values; no normalization means re-seeding would be identical to current state.
