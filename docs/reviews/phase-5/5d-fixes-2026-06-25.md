## Opus findings applied (2 total)

1. **settings.tsx: sleep validation ran after URL/token/model saves — partial save showed misleading 'Save failed'**
   - Moved ALL sleep field parsing and validation to BEFORE `setSaveState('saving')` and any async saves.
   - Added client-side cross-field check: `parsedThreshold >= parsedGood` → show targeted "Poor sleep threshold must be less than good sleep threshold" alert and return early.
   - Invalid sleep values now show a specific alert with zero side effects; the generic "Save failed" catch only fires for genuine network failures.

2. **health-connect.ts: no guard against totalMs <= 0 from degenerate HC sessions**
   - Added `if (totalMs <= 0) return null` after the session loop.
   - Degenerate records (where `endTime <= startTime`) no longer produce `durationHours = 0` and a false "0.0h Poor" card; they are treated as no-data.
