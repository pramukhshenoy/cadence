## Findings

### O1 — No tests for DELETE /complete endpoint [MEDIUM — MUST FIX]
File: `server/src/__tests__/habits.test.ts`
Issue: The new DELETE /:id/complete endpoint has zero test coverage. Jest mock for `habitCompletion` only has `upsert` — `deleteMany` is undefined, so any test hitting the route would throw. Missing cases: success 204, habit-not-found 404, auth guard, correct deleteMany arguments.
Fix: Add `deleteMany: jest.fn()` to mock; add describe block covering the above.

### O2 — Dashboard date math diverges from X-Timezone contract [LOW — track separately]
File: `app/src/app/index.tsx`, lines 33–45
Issue: `new Date(dueDate).setHours(0,0,0,0)` uses device's JS timezone, not the X-Timezone header. Pre-existing pattern from task-item.tsx. Can drift ±1 day at UTC-offset boundaries. Not a 2d-specific defect.
Fix: Out of scope for 2d. Track as future: emit `dueLocalDate` string from server or standardise comparison.

### O3 — No optimistic update; visible lag + double-tap risk [LOW — recommended]
File: `app/src/lib/habits.ts`, lines 18–38
Issue: Checkmark doesn't flip until round-trip completes; stale `completedToday` is readable between tap and refetch. Fast double-taps are idempotent (no data corruption) but UX feels broken over any latency.
Fix: Add `onMutate` optimistic update that cancels queries, flips `completedToday` in cache, rolls back on error.

### O4 — Streak label doesn't pluralize [LOW — recommended]
File: `app/src/components/habit-item.tsx`, line 43
Issue: "2 day streak" should be "2 days streak".
Fix: `${streak === 1 ? unit : unit + 's'} streak`

### O5 — weeklyTargetDays is a raw JSON string in client type [LOW — track separately]
File: `app/src/types/habit.ts`, line 8
Issue: Leaks server storage detail (JSON-encoded string) into client type. Harmless now; latent trap if future code reads it expecting `number[]`.
Fix: Out of scope for 2d. Track for future: parse server-side and type as `number[] | null`.
