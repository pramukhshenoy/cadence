# Phase 3b+3d — Review Fixes (2026-06-24)

## Sonnet Review Findings

### S1 — PATCH /api/settings used `update` instead of `upsert` (FIXED)
`prisma.settings.update` throws Prisma P2025 if the singleton row is absent.
**Fix:** Changed to `prisma.settings.upsert` in `server/src/routes/settings.ts`.

### S2 — Model-ID validation duplicated in two routes (FIXED)
`claude-`/`gpt-` prefix check was copy-pasted in both `chat.ts` and `settings.ts`.
**Fix:** Extracted `isValidModelId()` to `server/src/lib/models.ts`; both routes import it.

### S3 — Silent save skip when `selectedModel` is empty (FIXED)
`handleSave` skipped `updateModel.mutateAsync` when `selectedModel === ''` but still showed "Saved!".
**Fix:** Added `settingsLoading` to `isSaveDisabled` in `app/src/app/settings.tsx`.

## Opus Review Findings

### O1 — Five WHY-comments deleted from `chat.ts` (FIXED)
Non-obvious invariants lost: abort-before-async ordering, DB-before-flushHeaders window, Anthropic first-message constraint, 19+1=20 budget, atomic-persist rationale.
**Fix:** Restored all five comments in `server/src/routes/chat.ts`.

### O2 — `isSaveDisabled` omitted `modelsLoading` (FIXED)
Save was allowed while models were still loading, leaving `selectedModel === ''` and silently skipping the PATCH.
**Fix:** Added `modelsLoading` to `isSaveDisabled` in `app/src/app/settings.tsx`.

### O3 — `isValidModelId` used prefix-only check, allowing fake model IDs (FIXED)
`claude-doesnt-exist` would pass 400 validation, fail mid-stream (HTTP 200 already committed).
**Fix:** Changed to exact-match against `CLAUDE_MODELS` + `OPENAI_MODELS` arrays in `server/src/lib/models.ts`.
